## 1. はじめに

### 1.1 目的
本文書は、車両ステータス通知クライアント（以下「本クライアント」）のソフトウェア要件を定義する。
本クライアントは、SOME/IPプロトコルを介して車両ステータスサーバからデータを受信し、
表示・監視・アラート通知を行う。

### 1.2 スコープ
本クライアントは演習用アプリケーションであり、以下を対象範囲とする。
- SOME/IP Notificationの受信と表示
- SOME/IP Request/Responseによるデータ取得
- 閾値監視とアラート送信
- 受信データのバリデーション

### 1.3 参照文書

| 文書ID | 文書名 |
|--------|--------|
| IFS-VS-001 | インターフェース仕様書 (02文書内) |
| AUTOSAR_PRS_SOMEIPProtocol | SOME/IP Protocol Specification(ご参考でOKです) |

### 1.4 用語

| 用語 | 定義 |
|------|------|
| Notification | サーバからの周期的な一方向通知 |
| Request/Response | クライアントが要求しサーバが応答する同期通信 |
| DTC | Diagnostic Trouble Code（診断トラブルコード） |

## 2. 機能要件

### 2.1 Notification受信機能

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| SRS-FUNC-001 | クライアントはUDPポート30491で車両ステータスNotificationを受信できること | 必須 |
| SRS-FUNC-002 | 受信したNotificationのSOME/IPヘッダーを解析し、Service ID (0x1001) およびMethod ID (0x8001) を検証すること | 必須 |
| SRS-FUNC-003 | 受信ペイロード（12 bytes）をインターフェース仕様書に従いデシリアライズし、車両ステータスデータを復元すること | 必須 |
| SRS-FUNC-004 | 受信した車両ステータスをコンソールに表示すること。表示項目は車速、エンジン回転数、ギアポジション、ウィンカー状態、走行距離、燃料残量の6項目とする | 必須 |
| SRS-FUNC-005 | 表示は最新の受信データで上書き更新すること（ログ出力ではなくリアルタイム表示ですが、ログでもOK） | 推奨 |

### 2.2 Request/Response機能

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| SRS-FUNC-010 | クライアントはサーバに対しGET_VEHICLE_STATUS (Method ID: 0x0001) リクエストを送信し、レスポンスを受信できること | 必須 |
| SRS-FUNC-011 | クライアントはサーバに対しGET_DIAGNOSTIC_INFO (Method ID: 0x0002) リクエストを送信し、レスポンスを受信できること | 必須 |
| SRS-FUNC-012 | リクエスト送信後、3秒以内にレスポンスが得られない場合はタイムアウトとして処理すること | 必須 |
| SRS-FUNC-013 | レスポンスのReturn Codeを検証し、E_OK (0x00) 以外の場合はエラーログを出力すること | 必須 |

### 2.3 アラート通知機能

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| SRS-FUNC-020 | 車速が120 km/hを超過した場合、速度超過アラート (SPEED_WARNING) をサーバに送信すること | 必須 |
| SRS-FUNC-021 | エンジン回転数が7000 rpmを超過した場合、回転数超過アラート (RPM_WARNING) をサーバに送信すること | 必須 |
| SRS-FUNC-022 | 燃料残量が10.0%を下回った場合、燃料低下アラート (FUEL_LOW_WARNING) をサーバに送信すること | 必須 |
| SRS-FUNC-023 | アラートの重要度(severity)は以下の通りとする：速度130km/h超過またはRPM8000超過はCRITICAL(3)、それ以外はWARNING(2) | 必須 |
| SRS-FUNC-024 | 同一種別のアラートは最低1秒間隔を空けて送信すること（チャタリング防止） | 推奨 |

### 2.4 データバリデーション機能

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| SRS-FUNC-030 | 受信データの各フィールドが定義済みレンジ内であることを検証すること | 必須 |
| SRS-FUNC-031 | レンジ外のデータを受信した場合、警告ログを出力し、当該データの処理をスキップすること | 必須 |
| SRS-FUNC-032 | ペイロードサイズが期待値と異なる場合、エラーログを出力し、当該メッセージを破棄すること | 必須 |

## 3. 非機能要件

| 要件ID | 要件 | カテゴリ |
|--------|------|----------|
| SRS-NFR-001 | クライアントはPython 3.9以上で動作すること | 環境 |
| SRS-NFR-002 | 外部ライブラリへの依存なし（標準ライブラリのみ）で実装すること | 環境 |
| SRS-NFR-003 | 100ms周期のNotificationを取りこぼさず受信できること（同一ホスト内通信の条件下） | 性能 |

| SRS-NFR-005 | 起動時にバインドアドレスとポートをコンソールに表示すること | 運用 |

## 5. 制約・前提条件

- サーバは同一ホスト上で動作するものとする（127.0.0.1）
- サーバポートは30490、クライアントポートは30491を使用する
- UDP通信を使用する（TCP/SOME-IP-TPは対象外）
- Service Discovery (SOME/IP-SD) は本演習の対象外とする
